# 项目里程碑 (Milestones)

## 2026-01-14: RN 0.83 架构适配与静默化完成

### 核心成就
完成了 SoundTherapyPro 在 React Native 0.83 (Bridgeless 模式) 下的真机深度适配，确保了核心音频功能的稳定性与流畅性。

### 技术细节
1.  **音频引擎重构**:
    *   解决了 `AudioService` 在新架构下的 Promise 挂起死锁问题。
    *   引入了 `withTimeout` 超时保护机制，防止原生层无响应导致 UI 无限 Loading。
    *   优化了 `switchSoundscape` 切换链路，移除了所有临时的人为延迟（sleep），实现了音频切换的“秒开”体验。
2.  **体验静默化**:
    *   彻底清理了所有调试阶段的 `Alert.alert` 弹窗（Step 1-5），全部替换为 `console.debug` 结构化日志。
    *   用户在切换场景时不再受弹窗打断，体验回归沉浸式。
3.  **稳定性策略**:
    *   暂时冻结（注释）了 `ImmersivePlayerScreen.tsx` 中复杂的 `Animated.parallel` 动画逻辑。
    *   此举旨在排除 JS 线程抢占问题，优先保障音频指令的准确下达与执行。

### 当前状态
*   **音频**: 稳定，切换流畅，声音播放正常。
*   **UI**: 响应及时，无死锁，但部分视觉特效（呼吸、模糊渐变）处于暂存状态。

### 下一步计划
*   逐步恢复并优化 UI 动画，考虑使用 Reanimated 库以减轻 JS 线程负担。
*   进一步打磨沉浸式播放页面的视觉细节。

## 2026-01-18: 场景体系与沉浸体验稳定版

### 核心成就
在稳定音频内核的基础上，完成了场景体系、分类浏览与沉浸式播放体验的一轮成型，实现「可长期使用」的基础版本。

### 技术与功能要点
1. **场景模型统一化**
   * 使用 `baseAudios` 作为唯一音源清单，通过 `SCENES` 映射生成完整场景对象。
   * 为 `Scene` 增加 `category` 字段，并引入 `SceneCategory = 'Nature' | 'Healing' | 'Brainwave' | 'Life'`。
   * 为 19 个基础场景完成精细分类：自然环境、疗愈类、脑波类与生活环境。

2. **场景选择体验升级**
   * 在 `SoundscapeBottomSheet` 中增加分类切换条（自然 / 疗愈 / 脑波 / 生活）。
   * 支持按分类过滤场景列表，便于用户快速定位需要的声音类型。
   * 打开底部弹窗时，自动跳转到当前播放场景所在分类，并对该条目高亮 + 勾选。

3. **氛围点缀与混音体系**
   * `AmbientPickerSheet` 支持主音量 + 雨声 + 篝火三个维度的混音调节。
   * 使用 `react-native-sound` 作为氛围与交互音的播放引擎，并通过 `Image.resolveAssetSource` 解决静态资源在 iOS/Android 下的路径和类型差异。
   * 支持保存和恢复「混音方案」，包括场景 ID、主音量、雨声/火焰音量及当前氛围类型。

4. **交互音效修复与增强**
   * 修复了 `apple_crunch`、`match_strike` 在部分平台无声的 Bug（数字资源 ID 转 URI）。
   * 与播放/暂停按钮逻辑打通，在切换播放状态时增强反馈感（火柴点燃 / 咬苹果声）。

5. **计时与睡眠逻辑**
   * 完成睡眠定时器基础闭环：设置时长、倒计时显示、结束时淡出停止。
   * 当定时结束前 10 秒开启淡出（可控开关），避免突兀中断。
   * 为环形进度点引入统一进度源（音轨进度 / 定时剩余时间），避免 UI 与实际状态不一致。

6. **稳定性与代码结构**
   * `AudioService` 成为音频状态与定时器的单一数据源，对外提供订阅接口而非直接依赖 TrackPlayer hooks。
   * 所有场景与 FX 音效统一使用 RN 静态资源体系，避免路径大小写、扩展名差异导致的崩溃。

### 当前状态
* **主线功能**: 可稳定完成「选场景 → 播放 → 调音量/加氛围 → 设置定时 → 退出」的完整睡眠/放松流程。
* **音频体验**: 场景切换流畅、交互音与氛围音可稳定叠加，无明显卡顿或无声 Bug。
* **UI/交互**: 沉浸播放器视觉与动效已基本成型，底部场景选择与氛围面板可稳定使用。
