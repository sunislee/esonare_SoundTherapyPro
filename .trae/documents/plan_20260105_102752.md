# 修复计划

## 1. 动画逻辑修复 (Fix Animation Error)

### 问题分析
- `borderWidth`动画在Android原生驱动中不被支持

### 解决方案
- 确保所有涉及`borderWidth`或`shadow`的动画都设置`useNativeDriver: false`
- 验证现有动画配置，确保只有`opacity`和`transform`使用`useNativeDriver: true`

### 技术实现
```javascript
// 已正确配置，无需修改
Animated.spring(borderWidthAnim, {
  toValue: 1.5,
  friction: 8,
  tension: 40,
  useNativeDriver: false, // 正确
});
```

## 2. 移除实心圆盘 (Ditch the Solid Disc)

### 问题分析
- 当前多层背景光晕（auraLayer1-4）形成了遮挡中心的实心圆盘
- 透明度不够高，模糊效果不足

### 解决方案
- 移除多层光晕，使用单个极高透明度的氛围晕影
- 增加模糊效果，扩大尺寸，确保中心镂空
- 降低整体不透明度至0.1，使文字清晰可见

### 技术实现
```javascript
// 替换多层光晕为单个氛围晕影
<Animated.View 
  style={[
    styles.ambientGlow,
    {
      backgroundColor: getGlowColor(0),
      opacity: auraOpacity,
    }
  ]} 
/>
```

## 3. 实现外发光效果 (Outer Glow Instead of Fill)

### 问题分析
- 当前按钮选中效果包含内填色，质感差
- 颜色过于鲜艳，缺乏柔和感

### 解决方案
- 移除按钮内部填充色，仅保留边框和外阴影发光
- 使用柔和的半透明颜色（如雨夜使用rgba(100, 149, 237, 0.5)）
- 增强外阴影效果，减弱内发光

### 技术实现
```javascript
// 调整按钮样式
sceneButton: {
  // 移除内填充，仅保留边框
  backgroundColor: 'rgba(255, 255, 255, 0.05)',
  // 激活时外发光
  ...(isActive && {
    shadowColor: scene.softGlowColor,
    shadowOpacity: 0.5,
    shadowRadius: 20,
  }),
}
```

## 4. 布局微调 (Layout Adjustment)

### 问题分析
- 按钮离中心太近，视觉拥挤
- 标题需要确保在最顶层

### 解决方案
- 增大按钮圆环半径至`width * 0.65`，使按钮分布更靠外
- 确认标题`zIndex: 999`

### 技术实现
```javascript
// 增大圆环半径
const radius = width * 0.65;

// 确认标题zIndex
zIndex: 999,
```
