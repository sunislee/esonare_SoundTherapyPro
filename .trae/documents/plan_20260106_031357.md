# 重构音频加载逻辑

## 核心問題診斷
1. 路徑格式正確，但缺少 `contentType` 導致 Android 無法識別 m4a 格式
2. reset 和延遲時間可能不足 300ms
3. 需要更健壯的狀態管理

## 重構方案

### 1. 精準 SCENES 映射
```
雨夜: final_healing_rain.m4a
圍爐: fireplace.m4a  
深海: ocean.mp3
林間: forest.m4a
粉噪: pink_noise_zen.m4a
阿爾法: alpha_wave.m4a
雙耳: binaural_beat.mp3
```

### 2. 健壯加載流程
```javascript
// 停止 → reset → 500ms延遲 → add(with contentType) → play
await TrackPlayer.stop();
await TrackPlayer.reset();
await new Promise(r => setTimeout(r, 500)); // 500ms確保句柄釋放

// m4a 顯式指定 contentType: 'audio/mp4'
// mp3 使用 contentType: 'audio/mpeg'
```

### 3. 增量延遲機制
首次加載不加延遲，切換時增加 500ms 延遲

### 4. 清理腳本
```bash
cd android && ./gradlew clean && cd .. && rm -rf android/build
```
