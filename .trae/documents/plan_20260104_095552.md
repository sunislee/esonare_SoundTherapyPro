## 抢占式场景切换重构计划

### 核心目标
实现"新场景优先"的快速切换，消除延时感，让用户点击立即响应。

### 修改方案

#### 1. 重构 `switchScene` 函数
```javascript
const switchScene = async (index) => {
  // 0. 防御性检查：忽略无效点击
  if (index === currentSceneIndex && index !== 0) return;
  
  // 1. 暴力清除所有计时器（抢占式中断）
  if (fadeRef.current) {
    clearInterval(fadeRef.current);
    fadeRef.current = null;
  }
  stopAllTimers();
  
  // 2. 立即更新 UI（使用 requestAnimationFrame 确保最快时间片）
  const targetScene = SCENES[index];
  requestAnimationFrame(() => {
    setCurrentSceneIndex(index);
    fadeAnim.setValue(0);
    Animated.timing(fadeAnim, { toValue: 1, duration: 800, useNativeDriver: true }).start();
    setTimeLeft(0);
  });
  
  // 3. 极速切换音频（不等待淡出）
  try {
    await TrackPlayer.reset();
    await TrackPlayer.add({
      id: targetScene.id,
      url: targetScene.audio,
      title: targetScene.title
    });
    
    // 如果是粉噪，立即设置低音量并异步淡入
    if (targetScene.id === 'pink_noise') {
      await TrackPlayer.setVolume(0.2);
      // 异步执行淡入，不阻塞
      setTimeout(() => fadeInForPinkNoise(), 50);
    } else {
      await TrackPlayer.setVolume(volume);
    }
    
    await TrackPlayer.play();
    activatePsychologyPreset();
  } catch (error) {
    console.error('音频切换错误:', error);
    // 错误恢复
    try {
      await TrackPlayer.reset();
      await TrackPlayer.add({...});
      await TrackPlayer.play();
    } catch (e) {}
  }
};
```

#### 2. 优化 `togglePlayback` 函数
- 简化逻辑，暂停时直接停止，不需要淡出
- 确保立即响应

#### 3. 添加预加载优化（可选）
- 在组件初始化时预加载所有场景的音频
- 切换时直接使用已加载的轨道

### 预期效果
- ✅ 点击场景按钮 → UI 立即切换（无延迟）
- ✅ 旧场景音频立即停止（不等淡出）
- ✅ 新场景音频立即播放
- ✅ 粉噪场景音量异步爬升（不影响切换速度）

### 修改范围
- `switchScene` 函数（约70行）
- `togglePlayback` 函数（约20行）
- 无需新增文件
