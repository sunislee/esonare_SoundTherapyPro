## 重写计划：去模式化场景设计

### 1. 完全重写App.js
- **移除模式相关UI**：删除中间的Sound Mode文本和底部的声音模式按钮
- **统一场景数组**：创建包含所有场景的scenes数组，每个场景包含title、bg、type和params
- **简化状态管理**：移除currentFrequency和currentSoundMode状态
- **极简切换逻辑**：changeScene只需调用SuperpoweredModule.playScene()

### 2. 更新SuperpoweredModule
- **添加playScene方法**：接收scene.type和scene.params
- **实现stopAll逻辑**：接收到playScene时立即停止所有声音
- **按类型启动生成器**：根据scene.type开启唯一对应的音频生成器
- **确保音量设置**：toggleAudio时确保masterVolume设置为1.0

### 3. 具体实现

#### App.js
```javascript
// 统一场景数组
const scenes = [
  {
    title: '雨夜',
    bg: require('./assets/images/rain_bg.jpg'),
    type: 'nature',
    params: { frequency: 440 }
  },
  // 其他场景...
  {
    title: 'Alpha 波',
    bg: require('./assets/images/forest_bg_new.jpg'),
    type: 'alpha',
    params: { frequency: 440 }
  }
];

// 简化的changeScene
const changeScene = (scene) => {
  SuperpoweredModule.playScene(scene.type, scene.params);
  setCurrentScene(scene.title);
};

// 移除模式相关UI
```

#### SuperpoweredModule.java
```java
@ReactMethod
public void playScene(String type, ReadableMap params) {
  // 停止所有声音
  stopAll();
  
  // 根据type开启对应生成器
  switch (type) {
    case "alpha":
      startAlphaWave(params);
      break;
    // 其他类型...
  }
}

private void stopAll() {
  // 停止所有音频生成器
}
```

### 4. 测试与调试
- 确保toggleAudio点击后masterVolume设为1.0
- 验证每个场景切换时只有对应音频播放
- 检查场景背景和标题显示正确

- 所有场景平行显示，无模式区分
- 场景切换逻辑极简，只需一行调用

- 修复声音问题，确保各场景都有声音
