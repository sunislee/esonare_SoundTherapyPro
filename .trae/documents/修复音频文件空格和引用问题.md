## 修复音频文件引用问题

### 1. ⚠️ 文件名空格问题确认
**是的，文件名中的空格会导致路径引用失败！**

当前文件名：
- `Dream Drift.mp3` ❌（有空格）
- `Liquid Peace.m4a` ❌（有空格）

当前代码引用：
- `require('./assets/audio/tracks/lullaby_dream_drift.mp3')` ❌（不匹配）
- `require('./assets/audio/tracks/meditation_liquid_peace.m4a')` ❌（不匹配）

### 2. 解决方案（二选一）

**方案 A：重命名文件（推荐）**
将文件重命名为：
- `Dream Drift.mp3` → `lullaby_dream_drift.mp3`
- `Liquid Peace.m4a` → `meditation_liquid_peace.m4a`

**方案 B：修改代码引用**
如果不想重命名文件，需要修改代码中的引用为：
- `require('./assets/audio/tracks/Dream Drift.mp3')`
- `require('./assets/audio/tracks/Liquid Peace.m4a')`

### 3. 执行编译
完成文件重命名或代码修改后，执行 `npm run android` 重新编译

### 4. 建议
强烈建议使用**方案 A**，因为：
- React Native 资源系统对带空格的文件名支持不佳
- 下划线命名更符合开发规范
- 避免跨平台兼容性问题
