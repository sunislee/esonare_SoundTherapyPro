## 修复音频效果叠加问题

### 问题分析
当前应用的音频效果（阿尔法波、粉红噪音、双耳节拍）会同时播放，导致声音叠加。需要实现互斥逻辑，确保同一时间只有一个音频效果在播放。

### 修复方案

#### 1. 修改 Java 层（SuperpoweredModule.java）
- **添加 AudioMode 枚举**：定义 Alpha、PinkNoise、Binaural 三种模式
- **实现互斥逻辑**：确保同一时间只有一个音频生成器在运行
- **清空缓冲区**：切换模式时清空音频缓冲区，防止声音残留
- **更新 generateAudio() 方法**：根据当前模式选择对应的音频生成器

#### 2. 修改 JS 层（App.js）
- **更新 handleAudioEffectToggle 函数**：实现互斥逻辑，打开一个效果时自动关闭其他效果
- **同步 UI 状态**：确保 UI 开关状态与原生状态一致

### 具体修改

#### SuperpoweredModule.java
1. 添加 AudioMode 枚举：
```java
private enum AudioMode {
    OFF, ALPHA, PINK_NOISE, BINAURAL
}
private AudioMode currentAudioMode = AudioMode.OFF;
```

2. 更新 setAudioEffect 方法：
- 实现互斥逻辑
- 更新 currentAudioMode
- 关闭其他效果

3. 更新 generateAudio 方法：
- 根据 currentAudioMode 选择音频生成器
- 在切换模式时清空缓冲区

#### App.js
1. 更新 handleAudioEffectToggle 函数：
- 打开一个效果时，自动关闭其他效果的 UI 状态
- 更新原生模块的状态

### 预期效果
- 同一时间只有一个音频效果在播放
- 切换效果时没有声音残留
- UI 开关状态与原生状态同步
- 阿尔法波、粉红噪音、双耳节拍功能正常
